14. マジックバイト（ファイル先頭バイト）からの種類判定

← トップ
lib/file-utils.js の extFromMagic() はファイルの先頭数バイトから種類を推定します。 URL の拡張子だけでは判定できないファイル（拡張子なし、不正な拡張子）でも、 ダウンロード後の Content-Type や header bytes から正しく分類されるかを確認してください。 結果一覧の「種類」列にツールチップ「ファイル先頭バイトから判定」が出れば成功です。

拡張子と中身がミスマッチなファイル

  • mystery.bin — 中身は PDF (%PDF-1.4)
  • archive.dat — 中身は ZIP (PK\x03\x04)
  • no-extension — 拡張子なしの PNG

マジックバイトリスト（拡張機能が認識）

これらは 10. 全拡張子 ページに正しい拡張子で並んでいるので、参考までに対応表のみ:

  • %PDF (0x25 50 44 46) → pdf
  • PK.. (0x50 4B 03/05/07) → zip / docx / xlsx / pptx
  • .PNG (0x89 50 4E 47) → png
  • 0xFF D8 FF → jpg
  • GIF8 (0x47 49 46 38) → gif
  • RIFF...WEBP → webp
  • RIFF...WAVE → wav
  • RIFF...AVI → avi
  • ftyp (offset 4) → mp4
  • OggS (0x4F 67 67 53) → ogg
  • fLaC (0x66 4C 61 43) → flac
  • ID3 (0x49 44 33) / 0xFF E0 → mp3
  • Rar! (0x52 61 72 21) → rar
  • 7z\xBC\xAF (0x37 7A BC AF) → 7z
  • 0x1F 8B → gz
  • BM (0x42 4D) → bmp
  • BZh (0x42 5A 68) → bz2
  • 0xFD 37 7A 58 5A 00 → xz
  • MZ (0x4D 5A) → exe
  • 0x30 26 B2 75 (ASF header) → wmv / asf

確認ポイント

  1. mystery.bin をスキャン結果の「種類」列でホバー → 「ファイル先頭バイトから判定」ツールチップが出る
  2. ダウンロード後、ファイルが PDF として正しく開けることを確認
  3. archive.dat も同様。中身は ZIP として展開できる
